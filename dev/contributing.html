Use a Consistent coding StyleÔÉÅ

We use several language guidelines to increase code quality, readability, and usability.

Python guidelinesÔÉÅ

  • Linting: Please ensure your Python code doesn‚Äôt contain any errors by checking it with the flake8 python linter.

  • Formatting: Please format all your scripts using the black python formatter.

Markdown guidelinesÔÉÅ

  • Linting and formatting: Please ensure your markdown code contains no errors and is formatted according to the remark-lint style guidelines.

GitHub actionsÔÉÅ

The ROS Gazebo Gym package contains several GitHub actions, which check code changes against the language guidelines above. As a result, when the above guidelines are not met, you will receive an error/warning when you create a pull request. Some of these actions will make pull requests which you can use to fix some of these violations. For other errors/warnings, you are expected to handle them yourself before merging them into the main branch. If you think a coding guideline needs to be corrected or your code structure doesn‚Äôt allow you to respect the guideline, please state so in the pull request.

Pre-commit hooksÔÉÅ

The ROS Gazebo Gym package also contains several pre-commit hooks that enforce the guidelines above before committing. To enable these hooks, please:

  1. Install node.

  2. Install python.

  3. Run npm install . and pip install -r requirements/dev_requirements.txt to install husky and the required linters.

Release guidelinesÔÉÅ

Releases are handled automatically by the release-please-action. This action uses the release-please tool to create release pull requests based on the commits automatically. When writing commit messages, you must follow the Commit guidelines since this expects you to adhere to the conventional commits specifications. For more information on how this tool works, see the release-please documentation.
